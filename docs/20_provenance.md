# 20. 解析のトレース（Provenance）

## 概要：再現性とデータ管理

科学的再現性は現代の研究において不可欠な要素である。マイクロバイオーム解析は多数のパラメータと手順が絡み合うため、解析フローの記録が特に重要になる。QIIME 2 の Provenance システムは、すべての解析ステップを自動的に記録し、任意のアーティファクト（`.qza`/`.qzv`）からその生成に至るすべての処理を遡れる仕組みを提供する。

### メリット・デメリット

| 観点 | 内容 |
|------|------|
| **メリット** | 完全な再現性の保証 |
| | 不正なプラグインバージョンの自動検出 |
| | 論文査読・共同研究時の監査証跡（audit trail） |
| | コマンド履歴の自動生成（Provenance replay） |
| **デメリット** | アーティファクトのファイルサイズが増加する |
| | 一度作成したアーティファクトは内容を変更できない（immutable） |

## QIIME 2 View での Provenance 確認

> （実験医学 2020.1 月号より改変）

QIIME 2 で強化された解析パイプラインの視覚化の仕組みについて触れておく。

`.qzv` ファイルを [QIIME 2 View](https://view.qiime2.org/) にアップロードし、「Provenance」タブをクリックすると、解析の流れが DAG（有向非巡回グラフ）として表示される。

各ノード（灰色の四角形）は処理の内容とその処理に用いたデータ情報、エッジ（矢印）は解析の流れを表す。クリックすることでパネルが展開され、以下の情報が詳細に記述されている：

- 実施時間
- 入力ファイル
- 出力ファイル
- 各種パラメータ
- 使用したプラグインのバージョン

### 2025.10 以降の改善

- **エラー検出機能**: 既知の問題を持つプラグインバージョンで生成された結果がある場合、Provenance ビューで警告が表示される
- **Artifact プレビュー**: `.qza` ファイルも View で内容のプレビューとダウンロードリンクが表示される
- **検索機能**: ノードの検索に青いハイライトが付くようになった

このように QIIME 2 では、解析パイプラインが視覚化されることで、**解析結果を再現できる仕組み**が整えられている。

## Annotation 機能（Archive v7.0, 2025.4+）

アーティファクトに任意のメモ（アノテーション）を追加できる機能。バッチ番号・実験条件・処理メモなどを埋め込むことで、ファイル名に頼らずにアーティファクトに意味を持たせられる。

```bash
# アーティファクトにアノテーション（メモ）を追加
qiime tools annotation-create \
  --i-artifact table.qza \
  --p-name "experiment-note" \
  --p-value "Batch 1, 2026-03-25, control samples only" \
  --o-annotated-artifact table-annotated.qza

# アノテーションの一覧表示
qiime tools annotation-list \
  --i-artifact table-annotated.qza

# アノテーションの取得
qiime tools annotation-fetch \
  --i-artifact table-annotated.qza \
  --p-name "experiment-note"
```

> **Note**: アノテーションは `.qza` ファイル内部に格納されるため、ファイルをコピー・移動しても情報が保持される。

## Archive v7.1（2025.10）— 暗号署名

Archive バージョン 7.1 では、アーティファクトへの**暗号署名**が導入された。これにより：

- アーティファクトが改ざんされていないことを検証できる
- 署名鍵によって生成者を特定できる（将来的な機能）
- 信頼できるソースからのアーティファクトのみを解析に使用する仕組みを構築できる

---

## conda-env.yaml の自動記録

2025.4 以降、アーティファクト生成時に使用した conda 環境全体（インストール済みパッケージ一覧）が `conda-env.yaml` として Provenance 内に自動記録される。これにより：

- 数年後も同一環境を再現できる
- 使用したソフトウェアの正確なバージョンを論文の Methods に記載できる

---

## チェックサムのアップグレード（MD5 → SHA-512）

Archive v7.1 以降、アーティファクト内部ファイルのチェックサムが **MD5 から SHA-512** に変更された。SHA-512 はより強力な衝突耐性を持ち、アーティファクトの整合性検証の信頼性が向上している。

## Provenance Replay（Python API、2026.1+）

Python API の `Results` オブジェクトから、Provenance に基づいてコマンドを自動再生成する機能が追加された。

```python
from qiime2 import Artifact

# アーティファクトを読み込み
table = Artifact.load("table.qza")

# Provenance から再現コマンドを生成（Python API 形式）
replay = table.provenance.replay()
print(replay)
```

生成されたスクリプトを実行することで、同一の解析を別の環境・別のデータセットに対して再現できる。CLI コマンドとして出力する機能も開発中。
